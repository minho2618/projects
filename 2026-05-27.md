카카오톡 로그인 구현

  ### 1. 현재 인증 환경

  현재 프로젝트는 **세션 기반 인증(Session-based Authentication)**을 사용하고 있습니다.

  • 로그인에 성공하면 Spring Security가 생성한 세션 ID( JSESSIONID  쿠키)를 통해 브라우저가 로그인 상태를 유지합니다.
  • 따라서 카카오 로그인 후에도 JWT가 아닌 스프링 시큐리티 세션에 인증 정보를 직접 등록하는 방식으로 연동해야 기존
  시스템과 완벽히 호환됩니다.
  ──────
  ### 2. 구체적인 구현 흐름

  카카오 로그인 버튼을 클릭한 시점부터 로그인이 완료될 때까지의 흐름은 다음과 같습니다.

    sequenceDiagram
        actor User as 사용자
        participant Front as 브라우저 (login.html)
        participant Server as 백엔드 서버 (AuthController)
        participant Kakao as 카카오 API 서버

        User->>Front: '카카오톡으로 로그인' 클릭
        Front->>Kakao: 카카오 로그인 페이지로 이동
        User->>Kakao: 로그인 및 동의
        Kakao->>Server: 콜백 URL로 Redirect (인증 코드 전달)<br/>GET /api/auth/kakao?code=...
        Server->>Kakao: 인증 코드로 토큰 요청 (POST /oauth/token)
        Kakao-->>Server: Access Token 반환
        Server->>Kakao: Access Token으로 사용자 프로필 요청 (GET /v2/user/me)
        Kakao-->>Server: 프로필 정보 반환 (ID, 이메일, 닉네임)
        Server->>Server: DB 확인 및 연동/가입 처리
        Server->>Server: Spring Security Session에 사용자 등록
        Server->>Front: 메인 페이지로 리다이렉트 (로그인 상태 완료)
  ──────
  ### 3. 작업할 컴포넌트별 상세 내용

  #### ① 데이터베이스 및 엔티티 확장 ( User.java )

  • 카카오 회원을 고유하게 식별할 필드를 추가합니다.
      •  provider :  "LOCAL"  (일반 가입) 또는  "KAKAO"  (카카오 로그인)
      •  providerId : 카카오 측에서 제공하는 고유 회원번호 (숫자형 문자열)
  • 회원 연동 로직:
      • 만약 카카오 이메일이 기존 일반 가입 이메일과 일치하면, 두 계정을 연동( provider 와  providerId  설정)하여
      하나의 계정으로 일반/소셜 로그인 모두 가능하도록 설계합니다.
      • 신규 가입: 비밀번호 필드가 필수( NOT NULL )이므로 무작위 UUID 값을 임시 비밀번호로 암호화하여 저장하고,
      이메일 인증 절차 없이 즉시 활성화( Status.ACTIVE ) 상태로 가입 처리합니다.


  #### ② 카카오 API 연동 고도화 ( AuthService.java )

  • 현재 주석 처리된  kakaoOAuthLogin  메소드를 수정하여 세션 로그인 방식에 맞게 사용자 정보를 가입/조회하고  User
  객체를 반환하도록 변경합니다.
  • 카카오 API 명세서에 맞춰 기존에 잘못 명시된 파라미터명( redirect_url  ➡️  redirect_uri )을 수정합니다.

  #### ③ 컨트롤러 및 세션 등록 ( AuthController.java )

  •  /api/auth/kakao  주소로 들어온 요청에서 인증 코드를 꺼냅니다.
  •  AuthService 를 통해 카카오 회원 가입 또는 기존 회원을 조회합니다.
  • 조회된 회원 정보를 바탕으로 Spring Security의  UsernamePasswordAuthenticationToken 을 생성하고, 이를
  SecurityContext에 넣은 뒤 HTTP 세션에 저장합니다.
  • 모든 처리가 완료되면 메인 페이지( / )로 이동시킵니다.
